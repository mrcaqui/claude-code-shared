# ExecPlans

複雑な機能追加やリファクタリングでは、ExecPlanを作成して設計から実装まで管理する。
計画の作成・実装・更新を行う際は、必ず先に `skills/execplan-spec/SKILL.md` を読み、その仕様に従うこと。
計画ファイルは `./plans` ディレクトリに `dd-concise-name.md` の命名規則で配置する。

# タスク管理

機能実装やコード変更を行う際は、Tasks機能を使って進捗を管理・追跡する。作業を明確なステップに分解し、進行に応じてタスクのステータスを更新すること。

# その他

判断を仰ぐ際は "AskUserQuestion" を使用する。
計画ファイルを ./plans ディレクトリに作成する際は、ファイル名の先頭に連番（dd）を付け、内容を簡潔に表す名前を続けること。
